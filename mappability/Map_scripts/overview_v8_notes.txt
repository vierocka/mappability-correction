v8 STAR mapping parameters — notes

Simulation geometry
───────────────────
Fragment length : 300 bp  (= 2 × READ_LEN)
R1              : seq[pos : pos+150]            forward
R2              : revcomp(seq[pos+150 : pos+300])  reverse complement of second half
Loop range      : range(L - 300 + 1)            ~half as many pairs as v1–v7

Compared to v4_d / v7_f (fake PE):
  v1–v7  R2 = revcomp( seq[pos : pos+150] )    same window as R1, insert = 0
  v8     R2 = revcomp( seq[pos+150 : pos+300] ) opposite end of fragment, insert = 300 bp

Parameter review
────────────────
Parameter                v8_a (full genome)   v8_b (dedup CDS)    v8_c (MANE RNA)
─────────────────────────────────────────────────────────────────────────────────
--alignIntronMax 1       not set — correct    ADDED (fixed)       set — correct
                         genome needs intron  CDS has no introns  MANE is spliced
--outSAMmultNmax 1       set                  set                 set
--outFilterMultimapNmax  40                   40                  40
--alignMatesGapMax       default (unlimited)  default             default
                         correct for RNA-seq
--alignEndsType          default Local        default Local       default Local
                         matches real data    same                same

Notes
─────
- Local (soft-clipping) is correct for mappability simulation: matches how real
  data is processed. EndToEnd would artificially lower UF by failing reads with
  minor tip mismatches.
- alignMatesGapMax = 0 (unlimited) is appropriate for RNA-seq because introns
  can be very long; STAR discovers concordance from alignment positions.
- outFilterMultimapNmax 40 is consistent with v1–v7. STAR reports one alignment
  per read (outSAMmultNmax 1) but considers up to 40 loci for MAPQ scoring.
- MAPQ = 255 means uniquely mapped. UF = (MAPQ-255 R1 reads) / simulated R1 reads.
