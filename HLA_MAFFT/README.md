# HLA_MAFFT — sequence identity across HLA/APM paralogs

Multiple sequence alignments of HLA and antigen processing machinery (APM) sequences extracted from three reference sets. The alignments quantify within-family sequence identity and visualise the sequence divergence landscape that drives multimapping in RNA-seq.

## Why this matters

Multimapping at the APM locus is not uniform across genes or exons — it depends on how similar each sequence is to other members of the same region. MAFFT alignments make this explicit: regions of high sequence identity are the exact regions where RNA-seq reads will multimap regardless of reference or tool choice.

## Sequence sets aligned

Three sets were extracted from the APM/HLA region and aligned independently:

| File | Sequences | Source | Content |
|---|---|---|---|
| `HLA_MANE.fna` | 6 | MANE Select v1.5 | Full spliced transcripts for the canonical HLA/APM genes |
| `HLA_dedup_cds.fna` | 78 | GRCh38.p14 dedup CDS | All deduplicated CDS entries mapping to the HLA/APM region, including pseudogenes and paralogous sequences |
| `HLA_primaryChr.fna` | 20 | GRCh38.p14 primary chr | Gene body sequences from chromosome 6 only (no alt loci) |

The contrast between the three sets illustrates why reference choice matters: with only 6 MANE transcripts, few sequences compete; with 78 CDS entries, far more do.

## MAFFT alignments

Each set was aligned with MAFFT (see `run_mafft.sh`). Alignment outputs:

| File | Description |
|---|---|
| `HLA_MANE.mafft.fna` | MAFFT alignment of MANE transcripts |
| `HLA_dedup_cds.mafft.fna` | MAFFT alignment of deduplicated CDS sequences (alignment length 1,546 bp) |
| `HLA_primaryChr.mafft.fna` | MAFFT alignment of primary chromosome gene bodies (alignment length 18,560 bp) |
| `*.mafft.log` | MAFFT run logs |

Alignment lengths reflect the divergence within each set: the 6 MANE transcripts are shorter and more similar to each other, while the 20 primary chromosome gene bodies span a long genomic region with insertion gaps.
